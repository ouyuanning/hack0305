# 监听模块使用说明（混合方案）

监听模块用于**自动识别当前关注的 GitHub Issue**，在创建 Issue 时自动关联。

## 1. 4层智能检测（自动降级）

不再依赖 Chrome 调试模式，采用 4 层策略自动选择最佳方式：

| 策略 | 方式 | 说明 |
|------|------|------|
| 1 | Chrome CDP | Chrome 以 `--remote-debugging-port=9222` 启动时可用 |
| 2 | 系统窗口 | 从活动窗口标题提取（支持 Chrome/Safari/Firefox） |
| 3 | 剪贴板 | 检测剪贴板中的 GitHub Issue URL |
| 4 | 手动输入 | 交互式输入 Issue URL 或编号 |

**优势**：
- 不强制 Chrome 调试模式
- 支持普通启动的 Chrome、Safari、Firefox
- 剪贴板有 Issue URL 时也能识别
- 100% 可用（手动输入兜底）

## 3. 使用流程

1. **可选**：以调试模式启动 Chrome（优先使用 CDP）
   ```bash
   # macOS
   open -a "Google Chrome" --args --remote-debugging-port=9222
   ```

2. 在浏览器中打开要关联的 GitHub Issue

3. **或** 复制 Issue URL 到剪贴板

4. 在聊天或脚本中说「我要提 Bug，MO数据备份同步工具不好用」

5. AI 会自动尝试 4 种方式识别 Issue，生成草稿时关联

## 7. 范围限制

- **仅识别 GitHub Issue**：所有策略都只处理 GitHub Issue 的 URL 或编号，非 GitHub 网页（URL、标题、内容）**不会被获取或处理**。
- **内容来源**：Issue 详情（标题、标签、正文）来自 **GitHub API**，不是解析网页 HTML。
- **隐私**：设计上不收集用户在非 GitHub 网站上的浏览信息。
